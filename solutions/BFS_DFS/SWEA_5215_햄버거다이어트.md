# SWEA 5215 🍔 햄버거 다이어트 (Java)

## 느낀점
>
- 각 재료를 선택하거나 선택하지 않는 방식으로 모든 조합을 탐색하는 문제였다
- 두 번째 dfs 문제, 아직은 더 연습 필요

---

## 흐름
1. 테스트케이스 개수 `T` 입력
2. 재료 개수 `N`, 제한 칼로리 `L` 입력
3. 각 재료의 맛 점수와 칼로리를 배열에 저장
4. DFS로 현재 재료를 선택하는 경우와 선택하지 않는 경우를 모두 탐색
5. 칼로리 합이 제한을 넘으면 탐색 중단
6. 모든 재료를 확인하면 최대 점수 갱신
7. 최종 최대 점수 출력

---

## 코드

```java
import java.util.Scanner;

class Solution
{
    public static void main(String args[]) throws Exception
    {
        Scanner sc = new Scanner(System.in);
        int T;
        T=sc.nextInt();

        for(int test_case = 1; test_case <= T; test_case++)
        {
            int n = sc.nextInt();
            l = sc.nextInt();

            t = new int[n];
            k = new int[n];

            for(int i = 0 ; i < n ; i++) {
                t[i]=sc.nextInt();
                k[i]=sc.nextInt();
            }

            maxScore = 0;
            dfs(0,0,0);

            System.out.println("#" + test_case + " " + maxScore);
        }
    }

    static int l;
    static int maxScore;
    static int[] t;
    static int[] k;

    static void dfs(int index, int sumKcal, int sumScore) {
        if(sumKcal > l) {
            return;
        }

        if(sumScore > maxScore) {
            maxScore = sumScore;
        }

        if(index >= t.length) {
            return;
        }

        dfs(index+1, sumKcal + k[index], sumScore + t[index]);
        dfs(index+1, sumKcal, sumScore);
    }
}
```

## 질문 & 실수 정리

### 새로 배운 것

- DFS로 부분집합을 만들 때는 현재 원소를 `선택하는 경우`와 `선택하지 않는 경우`로 나누면 된다
- `dfs(index, sumKcal, sumScore)`는 현재 `index`번째 재료를 볼 차례이며, 지금까지의 칼로리 합과 점수 합을 들고 가는 구조다
- 칼로리 제한을 넘은 경우는 더 이상 탐색할 필요가 없으므로 `return`으로 가지치기할 수 있다

---

### 실수했던 부분

- `maxScore`를 테스트케이스마다 초기화하지 않았다
    - 이전 테스트케이스의 최대 점수가 다음 테스트케이스에 남을 수 있다
    - `dfs(0, 0, 0)` 호출 전에 `maxScore = 0`을 해줘야 한다

- 처음에는 5215번과 1244번의 DFS 차이가 헷갈렸다
    - 1244번은 두 자리를 교환하는 DFS
    - 5215번은 재료를 선택하거나 선택하지 않는 DFS

- 5215번도 넓게 보면 백트래킹이다
    - 다만 `sumKcal`, `sumScore`를 인자로 넘기기 때문에 별도의 원상복구 코드가 보이지 않을 뿐이다

---

## AI 코드 리뷰

### 개선할 점

1. `maxScore` 초기화 필요
    - `static` 변수이므로 테스트케이스마다 반드시 `maxScore = 0`을 해야 한다

2. 변수명 개선
    - `l` → `limit`
    - `t` → `scores`
    - `k` → `calories`
    - `sumKcal` → `calorieSum`
    - `sumScore` → `scoreSum`

3. 종료 조건 위치 정리
    - 현재 코드도 동작하지만, 보통은 `index == 재료개수`일 때 최대 점수를 갱신하는 방식이 더 명확하다

4. `n`도 전역 변수로 관리 가능
    - `t.length`를 써도 되지만, `n`을 전역으로 두면 의미가 더 분명하다

---

## AI 개선 코드

```java
import java.util.Scanner;

class Solution {
    static int n;
    static int limit;
    static int maxScore;
    static int[] scores;
    static int[] calories;

    public static void main(String args[]) throws Exception {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();

        for (int test_case = 1; test_case <= T; test_case++) {
            n = sc.nextInt();
            limit = sc.nextInt();

            scores = new int[n];
            calories = new int[n];

            for (int i = 0; i < n; i++) {
                scores[i] = sc.nextInt();
                calories[i] = sc.nextInt();
            }

            maxScore = 0;
            dfs(0, 0, 0);

            System.out.println("#" + test_case + " " + maxScore);
        }
    }

    static void dfs(int index, int calorieSum, int scoreSum) {
        if (calorieSum > limit) {
            return;
        }

        if (index == n) {
            maxScore = Math.max(maxScore, scoreSum);
            return;
        }

        dfs(index + 1, calorieSum + calories[index], scoreSum + scores[index]);
        dfs(index + 1, calorieSum, scoreSum);
    }
}
```
