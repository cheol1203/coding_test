# SWEA 1225 🔐 암호생성기 (Java)

## 느낀점
>
- 앞에서 값을 꺼내고 뒤에 다시 넣는 구조라 Queue를 사용하면 자연스럽게 풀 수 있었다
- `poll()`로 맨 앞 값을 꺼내고 `offer()`로 맨 뒤에 넣는 흐름을 연습할 수 있었다

---

## 흐름
1. 테스트케이스는 총 10개이므로 10번 반복
2. 테스트케이스 번호 `t` 입력
3. 숫자 8개를 Queue에 저장
4. `1`부터 `5`까지 감소시키는 작업을 반복
5. Queue의 맨 앞 값을 꺼냄
6. 현재 감소값만큼 뺌
7. 값이 `0 이하`면 `0`을 넣고 종료
8. 값이 양수면 다시 Queue 뒤에 넣음
9. 최종 Queue 상태를 출력

---

## 코드

```java
import java.util.Scanner;
import java.util.Queue;
import java.util.ArrayDeque;

class Solution
{
    public static void main(String args[]) throws Exception
    {
        Scanner sc = new Scanner(System.in);

        for(int test_case = 1; test_case <= 10; test_case++)
        {
            int t = sc.nextInt();

            Queue<Integer> queue = new ArrayDeque<>();

            for(int i = 0; i < 8; i++) {
                int num = sc.nextInt();
                queue.offer(num);
            }

            boolean boo = true;

            while(boo) {
                for(int i = 1; i <= 5; i++) {
                    int num = queue.poll();
                    num -= i;

                    if(num <= 0) {
                        num = 0;
                        queue.offer(num);
                        boo = false;
                        break;
                    }

                    queue.offer(num);
                }
            }

            System.out.print("#" + t + " ");

            for(int i = 0; i < 8; i++) {
                System.out.print(queue.poll() + " ");
            }

            System.out.println();
        }
    }
}
```

## 질문 & 실수 정리

### 새로 배운 것

- Queue는 먼저 들어온 값이 먼저 나가는 FIFO 구조이다
- `queue.poll()`은 맨 앞 값을 꺼내면서 제거한다
- `queue.offer(value)`는 값을 맨 뒤에 추가한다
- `ArrayDeque`는 Queue 구현체로 사용할 수 있다
- 반복 종료 조건이 내부 `for`문 안에 있을 때는 `boolean` 변수를 사용해서 바깥 `while`문까지 종료할 수 있다

---

### 실수했던 부분

- 이번 코드는 큰 실수 없이 잘 작성했다
- 다만 변수명 `boo`는 의미가 약해서 `isRunning` 또는 `isEnd`처럼 바꾸면 더 좋다


---

## AI 코드 리뷰


### 개선할 점

1. 변수명 개선
    - `boo` → `isRunning` 또는 `isFinished`
    - `num` → `current` 정도로 바꾸면 의미가 더 명확하다

2. 종료 조건을 더 직관적으로 바꿀 수 있음
    - `boolean boo = true`보다 `boolean isEnd = false`로 두고 `while(!isEnd)` 형태가 더 읽기 쉽다

3. 출력 방식 개선 가능
    - `StringBuilder`를 사용하면 출력 성능과 가독성이 좋아진다

---

## AI 개선 코드

```java
import java.util.Scanner;
import java.util.Queue;
import java.util.ArrayDeque;

class Solution {
    public static void main(String args[]) throws Exception {
        Scanner sc = new Scanner(System.in);

        for (int test_case = 1; test_case <= 10; test_case++) {
            int caseNumber = sc.nextInt();
            Queue<Integer> queue = new ArrayDeque<>();

            for (int i = 0; i < 8; i++) {
                queue.offer(sc.nextInt());
            }

            boolean isEnd = false;

            while (!isEnd) {
                for (int minus = 1; minus <= 5; minus++) {
                    int current = queue.poll() - minus;

                    if (current <= 0) {
                        queue.offer(0);
                        isEnd = true;
                        break;
                    }

                    queue.offer(current);
                }
            }

            StringBuilder sb = new StringBuilder();
            sb.append("#").append(caseNumber).append(" ");

            while (!queue.isEmpty()) {
                sb.append(queue.poll()).append(" ");
            }

            System.out.println(sb);
        }
    }
}
```
